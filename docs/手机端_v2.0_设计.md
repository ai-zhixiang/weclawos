# WeClaw 手机端 v2.0 设计方案

> 基于现有 Android 项目重构，核心原则：
> **AI 不做决策，AI 做检索。用户在决策链上。**
> App 不持有文件管理/相册功能，只做 AI 到系统能力的通道。

---

## 一、核心设计理念

```
用户说话 → AI 理解意图 → App 调系统能力 → 用户选择/确认 → AI 处理
     ↑                                                    |
     └──────────────── 展示结果 ──────────────────────────┘
```

- **用户一直在决策链上**，AI 只负责检索和理解
- 所有文件/照片选择走**系统原生 picker**，不构建任何替代 UI
- 一行多余的文件管理界面都不加

---

## 二、导航架构

### 2 Tab（简洁到底）

```
💬 对话                ⚙️ 设置
(唯一界面)              (账号/缓存/版本)
```

| Tab | 说明 |
|-----|------|
| 💬 对话 | 整个 App 的全部，输入框+消息列表 |
| ⚙️ 设置 | 账号退出、缓存清理、版本信息 |

**不需要：** 文件管理 Tab、相册 Tab、素材库 Tab

## 三、对话页 (ChatScreen) — 升级

### 输入栏改造

```
┌──────────────────────────────────────────┐
│   [ 输入消息...            ]  🎤  📎 ➤  │
│                                 │        │
│                          ┌─────┴─────┐  │
│                          │ 📷 拍照    │  │
│                          │ 🖼️ 选照片  │  │ ← 系统原生 picker
│                          │ 📄 选文件  │  │
│                          │ 📍 位置    │  │
│                          └───────────┘  │
└──────────────────────────────────────────┘
```

点击 📎 弹出底部 Sheet，每个选项调对应的系统 intent：
- **拍照** → `MediaStore.ACTION_IMAGE_CAPTURE`
- **选照片** → `ActivityResultContracts.PickVisualMedia`（系统 PhotoPicker）
- **选文件** → `ActivityResultContracts.OpenDocument`
- **位置** → 定位权限 → 获取坐标

### 新增：AI 可嵌入交互控件

AI 在回复中可以附带操作按钮，App 渲染为可点击组件：

```
┌──────────────────────────────────┐
│ 找到 3 张前天拍的照片            │ ← AI 文本
│  ┌──────────────────────────┐    │
│  │  📷 查看并选择照片        │    │ ← 点击 → 调系统 PhotoPicker
│  └──────────────────────────┘    │
│  选好后告诉我，我来帮你做嗨卡    │
└──────────────────────────────────┘
```

### 其他 UI 升级

| 项目 | 改进 |
|------|------|
| 语音输入 | 输入框旁 🎤，走 Android SpeechRecognizer |
| Markdown 渲染 | 回复中的代码、链接、列表格式化显示 |
| 消息时间戳 | 同组消息合并，跨日显示日期分隔 |
| 快捷操作栏 | 底部保留（🎵🎬📖📰✍️） |

## 四、App 能力清单（由 AI 指令触发）

| 能力 | 触发语句示例 | 实现方式 |
|------|-------------|----------|
| 📷 拍照 | "拍张照片" | `ACTION_IMAGE_CAPTURE` intent |
| 🖼️ 选照片 | "用前天公园的照片" | `PickVisualMedia` + 日期筛选 |
| 📄 选文件 | "上个月的会议记录" | `OpenDocument` + MIME 筛选 |
| 📎 多个文件 | "这几份合同" | `OpenMultipleDocuments` |
| 📍 位置 | "附近有什么" | `FusedLocationProviderClient` |
| 🌐 打开网页 | "打开这个链接" | WebView（现有） |
| 📤 分享 | "把这个发给他" | `ACTION_SEND` intent |

## 五、数据流

```
用户输入 ─→ ChatViewModel
                │
                ├─→ WebSocketManager → AI 服务
                │        ← AI 流式回复 + 操作指令
                │
                └─→ 解析指令 → 调系统能力
                               │
                     ┌─────────┴──────────┐
                     │ 拍照 | 选照片 | 选文件 │
                     └─────────┬──────────┘
                               │ 用户选择/确认
                               ↓
                           结果 → AI 处理 → 展示
```

```yaml
  textSecondary: "rgba(255,255,255,0.4)"
```
